### What Went Well

1. **Architectural patterns from Epic 1 held strong** — Zustand store conventions, Shell/Bridge/SDUI layer separation, WebSocket message format survived without rework. Every new store followed established patterns exactly.

2. **TDD discipline maintained** — Red-green-refactor cycle followed rigorously across all 6 stories. ~366 new tests added, zero regressions at any point.

3. **Code review caught real bugs** — 8 MEDIUM issues caught during adversarial reviews. Example: `Orb.tsx` missing `connectionStore` import (would have been runtime crash). All fixed before merge.

4. **Pragmatic dependency management** — Three deliberate fallbacks: built-in `Animated` API instead of `react-native-reanimated`, View layers instead of `expo-linear-gradient`, `module_card` type reuse instead of new `module_link` type.

5. **Chat streaming worked first try** — WebSocket protocol, chatStore, and ChatThread integrated cleanly. Solid foundation, though true token-by-token streaming still needed (see Action Items).

---

### What Could Be Better

1. **Story 2.5 → 2.5b pivot was costly** — ~1,000 lines written then deleted. Keyboard-driven crossfade transitions proved fragile cross-platform. Tab navigation (2.5b) was the right answer but came after a full story of exploration. A technical spike before full implementation would have caught this earlier.

2. **Story 2.4 PromptChips on wrong screen** — Components placed on Home tab (dashboard) instead of Chat tab. Required post-review correction by user. Root cause: ambiguous spec + no UX placement validation with user before dev.

3. **Incomplete test file documentation** — Edge test files created in Stories 2.3 and 2.4 but missing from File List sections. Recurring gap across multiple stories.

---

### Key Insights

- **Keyboard-driven state is fragile** — Implicit transitions (keyboard up = chat mode) are unpredictable cross-platform. Explicit user actions (tab tap) always more reliable for major UI modes.
- **Discriminated unions over booleans** — `AgentStatus = 'idle' | 'thinking' | 'discovering' | 'composing'` proved more expressive than `isLoading`. Adopted across all stores.
- **Backward-compatible defaults reduce ripple** — Story 2.2 changed `_build_module_prompt` → 9 test updates. Story 2.3 added `persona_content = None` → zero ripple.
- **Pragmatism and discipline are not opposites** — Staying disciplined (TDD, layer separation, reviews) while being pragmatic (fallback deps, scope pivots) produces quality software efficiently.

### New Story: Robust Chat Streaming & Session Recovery

**Rationale:** Story 2.1 delivered "First Light" streaming (full response in single delta). For First Light to feel production-ready, the chat experience must be robust:

- **True token-by-token streaming** — LLM provider yields tokens, backend forwards in real-time
- **Granular agent status** — user knows what the agent is doing (thinking, fetching APIs, composing)
- **Background resilience** — agent continues working when user leaves the app; response ready on return
- **Session recovery** — on app open, guaranteed sync of all missed messages and data
- **Graceful disconnection** — backend completes work even if WebSocket drops

**Dependencies:** Patterns established here (background sync, message acknowledgment) will be reused by 4.1 (cron refresh).

### Epic 4 Preparation Needs

1. **Research APScheduler or alternatives** for Python background tasks (before 4.1)
2. **Design module lifecycle state machine** — active → stale → dormant → archived (before 4.2)
3. **Plan soft delete migration** — `archived_at`, `restored_at` columns (before 4.3)
