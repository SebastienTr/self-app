### What Went Well

**1. Architectural Patterns from Epic 1 Held Strong**
Zustand store conventions, Shell/Bridge/SDUI layer separation, WebSocket message format — all survived Epic 2 without rework. Every new store (`chatStore`, `connectionStore` enhancements) followed established patterns exactly.

**2. TDD Discipline Maintained**
Red-green-refactor cycle followed rigorously across all 6 stories. Test count grew linearly. Zero regressions at any point during the epic.

**3. Code Review Culture Caught Real Bugs**
8 MEDIUM issues caught during adversarial reviews. Example: `Orb.tsx` missing `connectionStore` import (would have been runtime crash). AmbientBackground initial opacity mismatch (visual discontinuity). All fixed before merge.

**4. Pragmatic Dependency Management**
Three deliberate fallbacks instead of adding new deps:
- Built-in `Animated` API instead of `react-native-reanimated` (not installed)
- View layers instead of `expo-linear-gradient` (not in deps)
- `module_card` type reuse instead of new `module_link` type

**5. Design Token System Comprehensive**
All components styled from `tokens.ts`. No hardcoded values escaped review. Consistency enforced automatically.

**6. Documentation-Driven Development**
Detailed "Dev Notes" in story files (~1,500 lines per story) served as executable specifications. Agents followed patterns exactly, reducing iteration.

### What Could Be Better

**1. Story 2.5 → 2.5b Pivot Was Costly**
~1,000 lines of code written then deleted. Keyboard-driven UI transitions proved fragile. Tab navigation (2.5b) was the right answer but came after a full story of exploration.

**Root cause:** UX specification described a crossfade pattern; architecture review should have flagged keyboard-driven state as risky before implementation.

**Action:** For future stories with complex UI state, run a brief technical spike (1-2 tasks) before full implementation.

**2. Story 2.4 Component Placement Error**
PromptChips were initially placed on the Home tab (dashboard) instead of the Chat tab. Required a post-review correction to move them to the Chat screen.

**Root cause:** The story spec referenced "empty state on Home tab" but the user's intent was "empty state before first message in Chat." Ambiguity in spec + insufficient user validation before dev.

**Action:** For UX-facing stories, validate wireframe/component placement with user BEFORE starting dev. A 2-minute confirmation prevents a post-dev refactor.

**3. Async File I/O Pattern Not Standardized**
Story 2.2 introduced `asyncio.to_thread()` for SOUL.md reads. No canonical guidance existed. Could have been documented in Epic 1's architecture.

**Action:** Document async file I/O pattern in architecture.md as an ADR.

**4. Test File Documentation Gaps**
3 edge test files created in Story 2.3 but missing from the story's File List section. Similar gaps in 2.4.

**Action:** Include file list validation as a code review checklist item.

---

### Key Insights

**1. Keyboard-Driven State is Fragile**
Implicit state-driven transitions (keyboard up = chat mode) are unpredictable across platforms. Explicit user action (tab tap) is always more reliable for major UI mode changes.

**2. Discriminated Unions Over Booleans**
`AgentStatus = 'idle' | 'thinking' | 'discovering' | 'composing'` proved more expressive than `isLoading: boolean`. Pattern adopted across all stores successfully.

**3. Shell/Bridge Layer Separation is Enforceable**
By explicitly separating pure components (Shell) from store-reading components (Bridge), reviews caught violations early. This prevents accumulation of implicit dependencies.

**4. Backward-Compatible Defaults Reduce Ripple Effects**
Story 2.2 changed `_build_module_prompt` signature → 9 test sites updated. Story 2.3 added `persona_content: str | None = None` → zero ripple. Default parameters are the pragmatic path.

**5. Store-Per-Domain Scales Cleanly**
Each story added domain-specific state without cross-domain coupling. `chatStore`, `connectionStore`, `moduleStore` remain independent. Testing stays fast and isolated.

- **First Light milestone pressure:** These are the last 3 stories before First Light is complete. Quality bar must stay high.

## Conclusion

Epic 2 delivered a complete conversational shell with persistent agent identity, persona customization, and intuitive tab navigation. The pivot from crossfade (2.5) to tabs (2.5b) was the right call — cleaner architecture, simpler code, better UX.
