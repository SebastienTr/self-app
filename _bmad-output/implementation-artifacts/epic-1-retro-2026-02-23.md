## What Went Well

1. **Zod 4 native JSON Schema** — Eliminated `zod-to-json-schema` dependency, simplified cross-stack validation pipeline. Breakthrough discovery in story 1.1 that pays off for the entire project.

2. **`self.sh` as dev accelerator** — Evolved organically across stories (zombie cleanup, `--reset`, auto-fill pairing tokens). Enabled tight dev-test feedback loop on physical device.

3. **Physical device testing in parallel** — Compensated for mobile knowledge gaps. Caught real bugs (e.g., `crypto.randomUUID()` unavailable on Hermes) that no unit test would find.

4. **Consistent TDD from story 1.3 onward** — Red-green-refactor pattern respected, test count grew steadily to 994.

5. **Reusable patterns established** — Hybrid persistent queue (in-memory + SQLite), auth gate pattern (single `auth` message), delta sync protocol, Twilight design tokens.

6. **Architecture reflects team strengths** — Heavy logic on backend (Python), mobile is primarily rendering. Good fit for a Project Lead with stronger backend/infra experience.

---

## What Didn't Go Well

1. **snake_case / camelCase conversion** — Generated bugs in 3/7 stories. No documented convention for boundary mapping (WebSocket messages, DB columns, TypeScript types).

2. **Async enforcement (Python)** — Recurring issue in stories 1.2, 1.3, 1.5. Async/await conventions discovered ad-hoc rather than documented upfront.

3. **DB connection lifecycle** — Went from session-scoped (1.5) to per-request (1.6 revert). No clear doctrine, causing rework.

4. **Multi-pass fixes on complex stories** — Stories 1.3 (retry logic, timeout handling) and 1.5 (race condition in message dequeue) required multiple correction rounds.

5. **Mobile ecosystem knowledge gap** — Hermes runtime quirks, Expo-specific APIs (`expo-secure-store`), `EXPO_PUBLIC_*` env var inlining. Learning curve managed but present.

6. **Schema versioning** — `schema_version` defined in 1.1 but migration handling inconsistent in subsequent stories. Technical debt accumulated.

---

## Key Insights

1. **Dev tooling is a force multiplier** — `self.sh` investment paid off exponentially. Every story benefited from the infrastructure.

2. **Undocumented conventions become recurring bugs** — Case conversion and async patterns repeated across stories because they weren't codified.

3. **Physical device testing compensates for knowledge gaps** — Short feedback loops surface platform-specific issues early.

4. **Architecture decisions compound** — The Zod-first schema, UUID auth (no JWT), WebSocket-only patterns set up a clean foundation for everything that follows.
