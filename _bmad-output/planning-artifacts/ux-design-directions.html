Unified Stream

Strengths

  • Zero navigation to learn — just scroll
  • Metamorphosis is seamless — chat naturally becomes dashboard
  • Messaging mental model — WhatsApp with superpowers
  • Simplest to build (solo dev advantage)

Risks

  • Old messages push modules down — finding a module means scrolling
  • 25+ modules = very long scroll
  • No spatial organization (Seb can't group by category)
  • New messages at bottom push modules up constantly
DIRECTION B

Sheet + Canvas

Modules live on a scrollable canvas (the main view). The chat is a bottom sheet that slides up when needed. Day 1: sheet is full-screen (no modules yet). As modules appear, the sheet recedes. The canvas grows with the user.

Strengths

  • Natural metamorphosis — sheet shrinks as canvas grows
  • Modules are always spatially stable (not pushed by messages)
  • Chat is always available (pull up) but not intrusive
  • Canvas supports future organization (tabs, drag-and-drop)

Risks

  • Bottom sheet interaction can feel fiddly on mobile
  • Day 1 experience feels empty (modules above, chat below)
  • More complex to build than unified stream
  • Sheet state management adds engineering complexity
DIRECTION C

Tab Split

Two explicit tabs: Chat and Home. Chat is the conversation. Home is the module dashboard. Clean separation, clear mental model. Day 1 starts on Chat tab; as modules accumulate, Home becomes the default landing.

Strengths

  • Crystal clear mental model — chat here, modules there
  • Home tab is a true dashboard — spatial, organized
  • Easy to add future tabs (Settings, Memory, Genome)
  • Familiar iOS/Android pattern — zero learning curve

Risks

  • Context switching — must leave chat to see module result
  • Breaks the "conversation produces visible result" flow
  • Day 1 Home tab is empty — where does user start?
  • Metamorphosis is abrupt (tab switch), not progressive
DIRECTION D

Contextual Shift

One screen that reshapes itself. When no modules exist: full chat. When modules exist: dashboard with an inline chat input at bottom. Tapping a module opens a contextual conversation about that module. The app smoothly morphs between states based on content density.

Strengths

  • Truly progressive metamorphosis — one screen, continuous shift
  • Chat input always visible at bottom — no navigation needed
  • Module creation result visible immediately in context
  • Undo toast naturally fits the flow
  • Agent messages weave between modules naturally

Risks

  • Conversation history gets buried as modules grow
  • Full chat mode needed for extended conversations
  • Needs careful state management (when to show messages vs modules)
