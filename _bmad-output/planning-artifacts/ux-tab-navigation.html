Tab Navigation

Replacing auto-transitions with explicit user-controlled navigation.
Dashboard-first. Chat as a tool. Settings for later.
Why We're Changing
The current two-mode architecture uses automatic crossfade transitions triggered by keyboard state, timers, and agent status. This is overengineered and unpredictable.
Before: Auto-Transitions
7 triggers (keyboard, timer, agent status...)
~100 lines of transition logic
App decides which screen you see
"Where did my conversation go?"
Chat input always visible (even on dashboard)
No settings screen — PairingScreen is conditional
After: Tab Navigation
1 trigger: user taps a tab
Standard tab navigator
User decides which screen they see
Each tab preserves its scroll position
Chat input lives in Chat tab only
Settings is a real tab with room to grow
Information Architecture
Three tabs. Dashboard is home. Chat is the tool. Settings configures the app.
Tab Icon Label Role Default?
Home 📦 Home Module dashboard — scrollable list of SDUI module cards. This is where users spend most time. Yes
Chat 💬 Chat Conversation with the agent. Messages, inline module links, creation ceremony. Input bar at bottom. No
Settings ⚙️ Settings Backend pairing, app preferences. Replaces the conditional PairingScreen. No
Design principle: The tab bar is the only navigation mechanism. No auto-transitions, no timers, no keyboard-driven mode switches. The user is always in control.

Module Link — The Bridge
When the agent creates a module during chat, a compact link card appears inline. Tapping "voir" switches to the Home tab and scrolls to the module with a highlight animation.
(hover: #E8A84C)
PropertyValue
ComponentModuleLink
ContextChat tab only, after agent confirms module creation
ContentModule emoji + title (left) — "voir →" action (right)
Tap action1. Switch to Home tab
2. Scroll to target module
3. Play highlight animation (amber border pulse, 2 cycles)
Background#101C2C

Highlight animation: When the Home tab receives focus via a "voir" action, the target module card gets a temporary amber border (#E8A84C) with a pulsing glow effect. Other cards dim to 50% opacity. Animation runs 2 cycles (3s total), then all cards return to normal.
Tab Bar Specification
A thin, discreet bottom bar. Always visible. No surprises.
(icon + label + padding)

Badge (notifications)Circle on tab icon — amber for count, red for alert
AnimationNone. Instant tab switch. No slide, no fade.
Badge rules:
• Home tab badge — Shows count of new modules created since user last viewed Home tab
• Settings tab badge — Red "!" when not paired (required action)
• Chat tab — No badge (user initiates conversation, not the other way)
Navigation Flows
Every user journey mapped.
ScenarioFlow
First launch (not paired) App opens → Home tab (empty state) → Settings tab has red badge "!" → User taps Settings → Enters URL + token → Connects → Badge disappears → User goes to Chat or stays on Home
First launch (paired, 0 modules) App opens → Home tab (empty state with orb + "Ask Self to create one →") → User taps link or Chat tab → Types request → Module created → Module link appears → User taps "voir →" → Home tab with highlighted module
Return user (has modules) App opens → Home tab (module dashboard) → User browses modules → Taps Chat to ask something new → Returns to Home
Module creation from chat User sends message → Agent responds → Creation ceremony animation → Agent confirms → Module link card appears → Badge increments on Home tab → User taps "voir →" → Home tab, scrolled to module, highlight pulse
Background → Foreground App resumes on the same tab the user left. No mode evaluation. No transition.

The new architecture eliminates all auto-transition logic from App.tsx.

Implementation Notes
For the dev agent.
TopicGuidance
Tab navigator Use @react-navigation/bottom-tabs with createBottomTabNavigator. Expo Router alternative: file-based app/(tabs)/ layout. Choose based on current routing setup.
Tab bar styling Custom tabBar component using Twilight tokens. Background #0C1420, border #1E2E44. Use existing tokens.ts values.
"voir →" navigation Use navigation.navigate('Home', { highlightModuleId: id }). Home screen reads param, scrolls FlatList to index, applies highlight style.
Badge state Track newModulesSinceLastHomeVisit in moduleStore. Reset to 0 when Home tab gains focus.
Settings tab Absorb PairingScreen content into Settings. Remove conditional render from App.tsx. Show pairing form when not connected, connection info when connected.
Keyboard handling Only needed in Chat tab for input margin. Remove all keyboard → mode transition logic.
